## Overview

The 2-Minute Rule is a core principle from David Allen's Getting Things Done (GTD) methodology that provides a simple decision-making framework for processing tasks and emails. When reviewing any item, if the required action can be completed in two minutes or less, do it immediately rather than deferring, delegating, or organizing it.

## The Rule

**"If it takes less than two minutes, do it now."**

That's it. No exceptions, no special cases, no elaborate decision trees.

## Why It Works

### Reduces Overhead
- Writing down a task takes about 30 seconds
- Reviewing it later takes another 30 seconds  
- Deciding to do it takes 20 seconds
- Starting it requires context switching (~1 minute)
- **Total overhead**: ~2.5 minutes

If the task itself takes less than 2 minutes, the overhead of managing it exceeds the time to just do it.

### Prevents Accumulation
- Small tasks pile up quickly
- 10 two-minute tasks = 20 minutes of backlog
- Backlog creates psychological burden
- Immediate completion maintains clean slate

### Maintains Momentum
- No context switching cost later
- Keeps workflow moving
- Provides micro-accomplishments
- Prevents procrastination on small items

## When NOT to Use the 2-Minute Rule

### During Deep Work
- Don't interrupt focused work for 2-minute tasks
- Batch them for later processing
- The cost of context switching exceeds the rule's benefit

### In Meetings
- Don't multitask on 2-minute items during important discussions
- Note them for processing after the meeting
- Exception: If directly relevant to meeting topic

### When Energy is Wrong
- If you're in creative flow, don't break it
- If a 2-minute task requires different energy/mindset
- Save it for appropriate context

### Fake Urgency
- Don't let others' poor planning create your 2-minute "emergency"
- Ask if it truly needs immediate attention
- Batch non-urgent 2-minute requests

## Time Tracking Implications

### Don't Track Individual 2-Minute Tasks
- Overhead of tracking exceeds task duration
- Creates noise in time logs
- Interrupts workflow for data entry

### Do Track Batch Processing
- "Email processing: 30 minutes" (includes multiple 2-minute replies)
- "Admin tasks: 20 minutes" (batch of small items)
- "Inbox zero: 45 minutes" (clearing accumulated items)

### Note the Pattern
- If doing many 2-minute tasks in a domain, might indicate:
  - Need for better systems/automation
  - Unclear boundaries with others
  - Poor planning upstream
  - Opportunity for delegation

## Variations and Adaptations

### The 5-Minute Rule
- Used by some practitioners for less time-sensitive work
- Better for creative fields where tasks have longer minimum times
- Risk: Can derail focused work more easily

### The 1-Minute Rule
- Ultra-strict version for highly focused individuals
- Useful in high-interruption environments
- Forces more disciplined task management

### Context-Dependent Rules
- 2 minutes during admin time
- 30 seconds during deep work time
- 5 minutes during email processing blocks

## Implementation Strategy

### Step 1: Practice Estimation
- Start tracking how long "quick" tasks actually take
- Calibrate your 2-minute sense
- Most people overestimate initially

### Step 2: Create Processing Rituals
- **Morning**: Process overnight email/messages
- **Lunch**: Clear accumulated 2-minute items
- **End of day**: Final inbox processing

### Step 3: Batch the Borderline
- If unsure if it's under 2 minutes, batch it
- Process batches during designated time
- Use actual timing to calibrate

### Step 4: Protect Deep Work
- During focus blocks, collect 2-minute items
- Process them during next admin block
- Don't let the rule interrupt important work

## Common Mistakes

### Mistake #1: Death by a Thousand Cuts
**Problem**: Doing every 2-minute task immediately all day
**Solution**: Batch them into 2-3 processing sessions daily

### Mistake #2: Overestimating Speed
**Problem**: Thinking tasks take 2 minutes when they take 5-10
**Solution**: Time yourself on recurring tasks; adjust threshold

### Mistake #3: Ignoring Context
**Problem**: Switching contexts for 2-minute tasks in wrong domain
**Solution**: Only do 2-minute tasks in current context

### Mistake #4: No Boundaries
**Problem**: Letting others create unlimited 2-minute interruptions
**Solution**: Set office hours for 2-minute requests

### Mistake #5: Missing the Bigger Picture
**Problem**: Never questioning why so many 2-minute tasks exist
**Solution**: Quarterly review; automate or eliminate sources

## Integration with Other Methods

### Getting Things Done (GTD)
- Core component of GTD inbox processing
- Used during weekly review
- Applied when processing captured items

### Time Blocking
- Create "Admin Blocks" for batched 2-minute tasks
- Don't let 2-minute rule break time blocks
- Process during transition periods

### Pomodoro Technique
- Use breaks between Pomodoros for 2-minute tasks
- Don't interrupt 25-minute focus sessions
- Batch for end of Pomodoro cycle

### Inbox Zero
- 2-minute rule is key to achieving inbox zero
- Immediate action vs. file vs. defer decision
- Prevents email backlog

## Advanced Applications

### Delegation Decision
If task takes <2 minutes and only you can do it → Do it now
If task takes <2 minutes but anyone could do it → Delegate
If delegating takes >2 minutes → Just do it

### Automation Opportunity
If same 2-minute task repeats often → Automate it
Investment in automation pays off after ~10 repetitions

### Process Improvement
Many 2-minute tasks = upstream process problem
Redesign systems to prevent task generation
