## Awesome Time Tracking Data

**Category:** Time Tracking Overview Resources  
**Brand:** ever-works  
**Source:** https://github.com/ever-works/awesome-time-tracking-data  
**Used by:** Backend dataset for the timetrack.works “Awesome Time Tracking” directory  
**License:** CC-BY-SA-4.0

### Overview
Awesome Time Tracking Data is a machine-readable dataset that catalogs tools, software, methodologies, and practices related to time tracking. It is designed to serve as structured data for directories and applications, such as the timetrack.works Awesome Time Tracking directory.

### Features
- **Machine-readable dataset**
  - Data structured for programmatic consumption and integration.
- **Coverage of time tracking domain**
  - Includes time tracking tools and software.
  - Includes methodologies and best practices for time tracking.
  - Includes time tracking–related practices and approaches, not just apps.
- **Directory-focused structure**
  - Serves as the structured backend for the timetrack.works Awesome Time Tracking directory.
  - Organized into folders such as `data`, `markdown`, and `pages` to support static/dynamic site generation.
- **Configurable metadata**
  - Uses YAML configuration files (`categories.yml`, `collections.yml`, `config.yml`, `tags.yml`) to define:
    - Categories for time tracking resources.
    - Collections or groupings of related items.
    - Global configuration for consuming applications/sites.
    - Tags for classification and filtering of entries.
- **Markdown content support**
  - Additional descriptive content for resources is stored in markdown files, enabling human-readable pages backed by structured data.
- **Open data / open source friendly**
  - Hosted on GitHub for transparency, version control, and collaboration.
  - Licensed under CC-BY-SA-4.0, allowing reuse and adaptation with attribution and share-alike.
- **Versioned dataset**
  - Managed in a Git repository with commit history, enabling tracking of changes to time tracking entries over time.

### Typical Use Cases
- Powering or extending time tracking directories (like timetrack.works).
- Building comparison tools, catalogs, or search interfaces for time tracking software and methodologies.
- Research and analysis of the time tracking tools ecosystem using a structured dataset.
