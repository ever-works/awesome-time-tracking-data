## Overview

The Two-Minute Rule is a decision-making principle from David Allen's Getting Things Done (GTD) methodology. It states that if an action will take less than two minutes to complete, you should do it immediately when you first encounter it, rather than capturing it for later.

## Core Principle

When processing your inbox (email, tasks, physical items), ask: "Will this take less than two minutes?"
- **Yes**: Do it now
- **No**: Defer, delegate, or delete it

## Rationale

The time required to:
- Capture the task
- Store it in your system
- Review it later
- Context switch to complete it

...is often greater than just doing the task immediately.

## Applications

**Email Management**
- Quick replies (under 2 minutes) → Send immediately
- Longer responses → Schedule or add to task list

**Task Processing**
- Filing a document → Do now
- Making a quick phone call → Do now
- Sending a calendar invite → Do now

**Physical Items**
- Hanging up a coat → Do now
- Putting dishes in dishwasher → Do now

**Digital Tasks**
- Updating a calendar entry → Do now
- Sharing a link with colleague → Do now
- Approving a simple request → Do now

## Benefits

- **Prevents Accumulation**: Small tasks don't pile up
- **Reduces Decision Fatigue**: Clear decision criteria
- **Maintains Momentum**: Quick wins create positive energy
- **Keeps Systems Clean**: Inbox stays manageable
- **Saves Time**: Eliminates overhead of task management for small items
- **Reduces Stress**: Fewer lingering open loops

## Common Misunderstandings

**"Do everything that takes under 2 minutes"**
- WRONG: Only during processing time, not during focus work
- Don't interrupt deep work for two-minute tasks

**"Two minutes is exact"**
- It's a guideline, not a strict rule
- Adjust based on context (some use 30 seconds, others 5 minutes)

**"Do all small tasks immediately"**
- Only applies when processing inputs
- Batch similar two-minute tasks when possible

## Implementation Tips

1. **Set Processing Times**: Schedule specific times to process inputs
2. **Practice Estimation**: Get good at judging task duration
3. **Have Tools Ready**: Keep systems accessible for quick actions
4. **Batch When Possible**: Group similar quick tasks
5. **Adjust Threshold**: Find your optimal time threshold
6. **Protect Focus Time**: Don't apply during deep work
7. **Be Realistic**: Account for context switching time

## Related Productivity Concepts

**Touch It Once (TIO)**
- Handle each item only once
- Complements the Two-Minute Rule

**Inbox Zero**
- Process all inputs to empty
- Two-Minute Rule helps achieve this

**Batch Processing**
- Group similar tasks together
- Can combine with Two-Minute Rule

## Variations

- **30-Second Rule**: For faster processing
- **5-Minute Rule**: For those with more processing time
- **Context-Dependent**: Adjust based on current work mode

## When NOT to Apply

- During scheduled deep work or focus time
- When in the middle of important, complex work
- When the "quick" task requires significant context switching
- When batching the task with others would be more efficient
- During meetings or collaborative work
